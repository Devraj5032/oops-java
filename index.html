Classes & Objects
Understand the blueprint-instance relationship that forms the heart of OOP.

What is a Class?

A class is a blueprint or template that defines the structure and behavior of objects. It specifies what data (fields) an object will hold and what actions (methods) it can perform. Think of a class like a recipe — the recipe itself isn't food, but you use it to create food.

classfieldsmethods

What is an Object?

An object is a real, concrete instance created from a class using the new keyword. Each object has its own copy of the fields (state) but shares the class's methods. Multiple objects can be created from the same class, each with different data.

Constructors
Special methods that initialize objects when they are created with new.

What is a Constructor?

A constructor is a special method called automatically when you use new to create an object. It has the same name as the class and no return type. Its job is to set up the object's initial state. Java provides a default no-arg constructor if you write none.

📌
this() vs this.

this.name refers to the current object's field. this(name, age) calls another constructor in the same class — must be the very first statement.

⚠️
Copy Constructor

Java has no built-in copy constructor (unlike C++). You can write one manually: Person(Person other) { this.name = other.name; }

Encapsulation
Protect data by hiding it inside the class and controlling access through methods.

What is Encapsulation?

Encapsulation bundles data (fields) and the methods that operate on that data into one unit, and restricts direct access to the data from outside. Fields are made private and accessed through public getter/setter methods, allowing validation before any change is applied.

Without encapsulation anyone can set balance = -99999 directly. With it, every change goes through validated methods, so the data stays consistent and correct.

Inheritance
Reuse and extend existing classes using the extends keyword.

What is Inheritance?

Inheritance lets a child class (subclass) acquire fields and methods from a parent class (superclass) via extends. It promotes code reuse and creates an IS-A relationship. Java allows only single inheritance — one parent class — but a class can implement multiple interfaces.

What is Polymorphism?

Polymorphism means "many forms." Java provides it in two ways: compile-time (method overloading — compiler picks which method) and runtime (method overriding — JVM picks which method based on the actual object's type at execution time).

Abstraction
Hide complexity. Show only the essential using abstract classes.

What is Abstraction?

Abstraction hides internal implementation details and exposes only what is necessary. Java provides it via abstract classes (partial abstraction — can mix abstract and concrete methods) and interfaces (full abstraction). An abstract class cannot be instantiated directly.

abstractcannot instantiatemust override

⚠️
Abstract class rules

If a class has even ONE abstract method the class must be declared abstract. Every subclass must implement all abstract methods — or also be declared abstract.

Advanced Abstraction
Interfaces
Pure contracts — the key to multiple inheritance and flexible design.

What is an Interface?

An interface is a completely abstract type defining a contract — method signatures that implementing classes must provide. Unlike abstract classes, a class can implement multiple interfaces. All methods are implicitly public abstract and all fields are public static final.

Rule of thumb

Use abstract class when classes share implementation (e.g., all animals breathe). Use interface for capabilities that unrelated classes might share (both Duck and Airplane can fly).
